## Issues
1. Install or update packages script, run as part of bootstrap, pauses on interactive install prompts. The occurred for Kea and Emacs. Controls did not work while script was running and had to kill it.
2. Public key auth was not working for either the transfer or the deploy user. It kept failing authentication verified by both the logs on github and journalctl on the router.
3. TPM was failing to match with the hash of the public key stored on the router.
4. Signature verification was failing on the router.
5. The deploy user was not able to run the install packages script.

## Resolutions
1. Manually installed kea and emacs to handle interactive prompts.Added DEBIAN_FRONTEND=noninteractive to install-or-update-packages.sh to suppress all debconf prompts on future runs. Kea starts unconfigured which is correct — full configuration comes in a later step.
2. The transfer and deploy users were modified to use /bin/bash as their shell. The nologin shell was printing output to stdout before ForceCommand could execute which interfered with SCP protocol negotiation. Security is maintained through two independent layers — ForceCommand in sshd_config and command= restriction in authorized_keys — neither of which depends on the nologin shell to function.
3. tpm2_nvread returns raw binary containing null bytes. Bash command substitution cannot store null bytes in variables and printed a warning to stderr. Fixed by piping tpm2_nvread output directly through xxd without storing raw binary in a bash variable, converting to hex string in a single pipeline.
4. It was determined that since the key was generated via openssl, that the signature should be verified via openssl instead of the verifysignature functionality of the TPM library. This change resolved that issue and the signature was verified.
5. The pipeline was updated to set the scripts in the repository to executable.

## Decisions Made
- Packages with interactive debconf prompts will be installed manually on first bootstrap until DEBIAN_FRONTEND=noninteractive is confirmed working. Script updated to prevent recurrence on future bootstraps.
