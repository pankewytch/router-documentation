## Issues
1. Kea was not binding to the infra interface after deployment and was not
   issuing DHCP leases on the infra VLAN despite the interface being present.
2. Home VLAN isolation needed explicit firewall rules to prevent cross-VLAN
   access between home and infra.

## Resolutions
1. Restarting Kea after the network interfaces were fully up resolved the
   binding issue. Root cause: Kea was started or reloaded during deployment
   before systemd-networkd had fully brought up the VLAN subinterfaces, so
   it could not bind its raw socket to the infra interface. Fixed by adding
   `networkctl wait --timeout=30 infra home` to deploy-network.sh so VLAN
   interfaces are confirmed ready before the deploy script exits and
   subsequent service restarts occur.
2. Added explicit inter-VLAN reject rules to the forward chain using
   `reject with icmpx type admin-prohibited` — clients on home receive an
   immediate ICMP admin-prohibited response rather than timing out, which
   is better UX and clearer debugging signal. Verified working by attempting
   to ping Eris and SSH into Sol from the home VLAN interface on Mac —
   both correctly rejected.

## Decisions Made
- VLAN isolation uses `reject` with explicit ICMP error rather than silent
  `drop` for inter-VLAN traffic — clients get immediate feedback that access
  is denied rather than waiting for a timeout. DMZ will use `drop` when
  configured since external-facing traffic should never get a response.
- DMZ is explicitly excluded from the LANS variable by comment in both
  tables — DMZ must be isolated from all internal VLANs and must never
  be grouped with trusted internal interfaces.
- Anti-spoofing on WAN interface is written and commented out, ready to
  enable when Sol connects directly to the public internet. Keeping it
  in the config as a comment ensures it is not forgotten.
- Bootstrap procedure documented: new Ubiquiti devices must be plugged into
  a dedicated infra access port, adopted by the controller, have IP Settings
  set to infra via the controller, then moved to a trunk port. This is the
  repeatable procedure that works reliably.
