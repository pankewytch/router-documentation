## Issues
1. Postgres Docker secret password file permissions were not explicitly set, so Postgres could not read them during initialization.
2. `kea-admin db-init` failed because `psql` was not in path.
3. Reservation upsert script used `.[]` instead of `.reservations[]`.
4. `jq` failed to parse `hw-address` due to `-` in key name.
5. `sed` replacement failed when Postgres password contained `/`.
6. Postgres `ON CONFLICT` clause lacked uniqueness due to incomplete conflict target.
7. Some `sed` and `jq` errors appeared only in GitHub Actions logs, not router journal.
8. DDNS failed with `RCODE 2 (SERVFAIL)` due to Bind9 rejecting updates.
9. Bind9 journal file could not be created under `/etc/bind/zones/`.
10. DHCP handshake failed when Kea listened on both parent and infra subinterface.
11. Ubiquiti devices sent management traffic as VLAN 1 after factory reset when controller pushed VLAN 100 too early.
12. Sol-Trunk native VLAN set to Infra (100) made switch unreachable.
13. Unifi switch IP settings set to VLAN 100 made switch unreachable.
14. Same IP (`10.10.100.1/23`) on parent and infra interfaces caused routing confusion.
15. Stale VLAN interfaces accumulated and persisted across `systemd-networkd` restarts.

## Resolutions
1. Set password file permissions to `644`; Postgres container then read secrets correctly.
2. Installed `postgresql-client` locally on Sol so `psql` exists for `kea-admin` migrations.
3. Changed iterator from `.[]` to `.reservations[]`.
4. Used `."hw-address"` in `jq` to treat key as a string.
5. Changed `sed` delimiter from `/` to `|` for password safety.
6. Added `subnet_id` to `ON CONFLICT` target to match unique constraint.
7. Added `exec > >(logger -t deploy) 2>&1` to deploy script so all output lands in journal.
8. Added `allow-update { 127.0.0.1; }` to all Bind9 zones to resolve `RCODE 2`.
9. Moved zone files from `/etc/bind/zones/` to `/var/lib/bind/zones/` to comply with AppArmor.
10. Removed `enp2s0f0np0` from Kea `interfaces-config`; Kea now listens only on infra.
11. Addressed VLAN fallback by moving infra to VLAN 1, aligning with Ubiquiti defaults.
12. Eliminated tagged/untagged asymmetry by moving infra to VLAN 1.
13. Avoided Unifi switch IP settings for management VLAN; rely on trunk native VLAN routing.
14. Removed same-IP dual-interface pattern; parent has no IP and infra subinterface has single authoritative IP.
15. Added cleanup function in `deploy-network.sh` to remove stale VLAN interfaces before redeploy.

## Decisions Made
- Postgres uses named Docker volume; reservations remain redeployable from source JSON in milliseconds.
- Plaintext password files are temporary while router remains behind existing network; will migrate to `systemd-cred` once stable.
- Adopted DDNS via `kea-dhcp-ddns` for dynamic hostname registration.
- Moved Bind9 zones to `/var/lib/bind/zones/` instead of modifying AppArmor profiles.
- Renumbered infra from VLAN 100 to VLAN 1 (`10.10.1.0/23`) to eliminate bootstrapping paradox permanently.
- Parent interface `enp2s0f0np0` will carry no IP in final design; all routing occurs on named VLAN subinterfaces.
