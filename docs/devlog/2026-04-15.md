## Accomplished

- Assigned a static IP to the router on my current network making it easier to access during development

## Issues
1. The router has coreboot as the BIOS and 2 com ports, COM0 on an RJ45 and COM2 on a USB C port. After creating the bootable flash drive, the router was plugged into my computer and accessed in the terminal via that console connection before being powered on. Once powered on, the BIOS was navigated until the drive was found and selected as the boot device. This did not go as planned because as soon as the EFI stubs were printed, the console connection was dropped. This was determined because although passwords are randomly generated each time for the installer and ubuntu users, an ssh connection was established with both users although the log in could not be completed due to the randomized passwords during install. In order to mitigate this, the grub terminal was accessed to determine all the serial ports and as it turns out, COM1 was on TTYS1 and not the default of TTYS0. Once that change was made in the boot command, the installation was displayed on the terminal until the cloud-init step where the menus to select language could not be displayed. It rendered as a sequence of wingdings.

## Resolutions
1. A monitor (via an HDMI cable) and a USB keyboard where directly attached to the monitor in order to load the Ubuntu OS as there were clearly issues with how the serial connection via the console was handling the Ubuntu installer when it came to the various navigable portions of the install. The COM1 port was tested after the install and there were no issues.

## Decisions Made
- Load the OS with a direct connection to the router and not via the console port
